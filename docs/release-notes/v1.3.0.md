# Ghost in the Droid v1.3.0

_Release candidate — target ship: TBD_
_Status: RC ready. Blocker list on final line._

## In one line

Ghost 1.3 turns Ghost in the Droid into a full **agent harness for Android** — six LLM backends, on-device Gemma, LangChain/LlamaIndex integration, and honest AndroidWorld numbers. 35 MCP tools (up from 28), same skill engine — swap the brain, keep the body.

## Highlights

- 🧠 **6 LLM backends** — Claude Code, Anthropic API, OpenRouter, Ollama, on-device Gemma, vLLM
- 📱 **On-device inference** ⭐ — Gemma runs inside the Android app itself via MediaPipe (`.task`) or llama.cpp (`.gguf`). Airplane-mode works.
- 🔗 **LangChain + LlamaIndex integration** — an Android body for your existing agent
- 🔐 **`android-agent login` — Claude subscription sign-in** (no API key). Wraps the sanctioned `claude` CLI auth; `doctor` verifies it's live. Gemini OAuth device-flow follows in v1.3.1.
- 🚀 **`uvx ghost-in-the-droid up`** — one-command install + boot; `doctor` command for preflight
- 🧨 **Batch-flow MCP primitive** — one round-trip for multi-step workflows (huge token/latency win)
- 🐳 **Docker+KVM emulator backend** — replaces broken native AVD path
- 🔍 **Tracing tab** — per-turn traces, token accounting, tool-call visibility
- 🌐 **Web search tool** — agent can search mid-conversation
- 📊 **Observability layer** — structured tracing across agent chat + tool execution, schema-adapter scaffold, dashboard viewer
- 📖 **How Ghost compares** doc + MCP client compatibility matrix — clear positioning vs LangChain Agents / OpenAI Agents SDK / Anthropic Computer Use
- 🔌 **Marketing jobs seam** — `POST /api/marketing-jobs/enqueue` for external orchestrators
- 🧭 **`lookup_lead` + `list_unread_leads`** MCP tools
- 🎯 **Clean AndroidWorld submission** — score TBD, unmodified upstream harness, submitted trajectories

## Fixes worth calling out

- 🔴 **On-device tool args** now handle both flat and nested shapes (was silently dropping args from Gemma's trained shape)
- 🔴 **`Device.adb` surfaces errors** — was returning stdout with `check=False`, letting ~38 MCP tools inherit phantom-success
- 🔴 **Batch-flow + adapter allow-list is fail-closed** — new/unknown tools are refused, not auto-allowed. `run_flow`, LangChain toolkit, LlamaIndex toolspec share **one** frozenset (`SAFE_DEVICE_TOOLS === FLOW_ALLOWED_TOOLS`), so they can't drift.
- 🔴 **`android-agent up` defaults to `127.0.0.1`** with an explicit `--host 0.0.0.0` opt-in that prints a network-exposure warning (was LAN-reachable without auth)
- 🟡 **`stop_agent` respects session boundaries** — was globally killing every claude subprocess
- 🟡 **Test suite is honest** — removed fixtures that wiped the real dev DB
- 🟡 **Concurrent-pytest DB isolation** — per-worktree DB path (`/tmp/gitd_pytest_<hash>.db`) prevents cross-branch test races
- 🟡 **`doctor` handles macOS Keychain-stored Claude creds** — Linux keeps the fast file-path; macOS falls back to `claude auth status` probe (storage-agnostic, no OS branching)
- 🟡 **`list_crashes` / `get_crash` surface ADB errors properly** — was silently returning "no crashes" on an offline device; also widened to `-b events` for ANRs
- 🟡 **Three `tap-element` code paths consolidated** — indexes now shared across surfaces
- 🟡 **OpenRouter provider** now has the full multi-turn tool-use loop

## Security

This release includes the first community-contributed security patch.

- 🔴 **CORS/CWE-352 fix** ([#10](https://github.com/ghost-in-the-droid/android-agent/pull/10), thanks [@sebastiondev](https://github.com/sebastiondev) via [Sebastion AI](https://github.com/apps/sebastionai)) — `allow_origins=["*"]` + `allow_credentials=True` caused Starlette's `CORSMiddleware` to reflect any request Origin. Fixed with an allowlist of localhost dev origins + `GITD_CORS_ORIGINS` env override.
- 🟡 **Known: `POST /api/skills/install` remains unauthenticated.** CORS closes the browser drive-by; any non-browser client with network access can still reach it. Mitigation: `up` now binds `127.0.0.1` by default (this release). Full closure — auth on sensitive endpoints + skill sandbox — is tracked as the top P0 for v1.3.1.
- 🟢 **Run behind a reverse proxy with auth** for any non-loopback exposure. `SECURITY.md` details.
