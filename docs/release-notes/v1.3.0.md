# Ghost in the Droid v1.3.0

_Release candidate — target ship: TBD_
_Status: DRAFT — updated as features land_

## In one line

Ghost 1.3 turns Ghost in the Droid into a full **agent harness for Android** — six LLM backends, on-device Gemma, LangChain/LlamaIndex integration, and honest AndroidWorld numbers. Same 38 MCP tools, same skill engine — swap the brain, keep the body.

## Highlights

- 🧠 **6 LLM backends** — Claude Code, Anthropic API, OpenRouter, Ollama, on-device Gemma, vLLM
- 📱 **On-device inference** ⭐ — Gemma runs inside the Android app itself via MediaPipe (`.task`) or llama.cpp (`.gguf`). Airplane-mode works.
- 🔗 **LangChain + LlamaIndex integration** — an Android body for your existing agent
- 🔐 **OAuth device-flow login** — ride your Claude Max / Gemini Pro subscription, no API key needed
- 🚀 **`uvx ghost-in-the-droid up`** — one-command install + boot; `doctor` command for preflight
- 🧨 **Batch-flow MCP primitive** — one round-trip for multi-step workflows (huge token/latency win)
- 🐳 **Docker+KVM emulator backend** — replaces broken native AVD path
- 🔍 **Tracing tab** — per-turn traces, token accounting, tool-call visibility
- 🌐 **Web search tool** — agent can search mid-conversation
- 🔌 **Marketing jobs seam** — `POST /api/marketing-jobs/enqueue` for external orchestrators
- 🧭 **`lookup_lead` + `list_unread_leads`** MCP tools
- 🎯 **Clean AndroidWorld submission** — score TBD, unmodified upstream harness, submitted trajectories

## Backends supported

| Provider | Runs where | Best for |
|---|---|---|
| [Claude Code](https://claude.com/claude-code) | Your machine (CLI) | Free daily driver |
| [Anthropic API](https://docs.anthropic.com/) | Anthropic cloud | Best-in-class quality |
| [OpenRouter](https://openrouter.ai/) | Cloud multiplexer | Any model, one key |
| [Ollama](https://ollama.com/) | Your machine | Fully local, unlimited |
| **On-device** ⭐ | The phone itself | 100% offline, private |
| [vLLM](https://docs.vllm.ai/) | Your GPU | Self-hosted full-precision |

## Ecosystem integration (new)

Ghost can now plug directly into the two biggest agent frameworks:

- **LangChain**: `GhostToolkit` for `BaseTool`-shaped consumption
- **LlamaIndex**: `ToolSpec` adapter

Or point any MCP-compatible agent at Ghost's MCP endpoint. See [ghostinthedroid.com/integrations](https://ghostinthedroid.com/integrations/).

## Fixes worth calling out

- 🔴 **On-device tool args** now handle both flat and nested shapes (was silently dropping args from Gemma's trained shape)
- 🔴 **`Device.adb` surfaces errors** — was returning stdout with `check=False`, letting ~38 MCP tools inherit phantom-success
- 🟡 **`stop_agent` respects session boundaries** — was globally killing every claude subprocess
- 🟡 **Test suite is honest** — removed fixtures that wiped the real dev DB
- 🟡 **Three `tap-element` code paths consolidated** — indexes now shared across surfaces
- 🟡 **OpenRouter provider** now has the full multi-turn tool-use loop

## Cleanup

- Moved premium schema out of `gitd/db.py` into the private plugin
- Deleted ~180 LOC of dead provider code (`_chat_claude_code`)
- CORS + input validation hardened across POST endpoints

## API surface

**New CLI:**
- `android-agent up` — boot server + dashboard
- `android-agent doctor` — preflight check (adb, ports, deps, keys, migrations)
- `android-agent login` — OAuth device-flow for Anthropic / Gemini

**New endpoints:**
- `POST /api/marketing-jobs/enqueue` — external orchestrator hook

**New MCP tools:**
- `run_flow` / `batch` — server-side batched execution (with injection blocklist)
- `lookup_lead`, `list_unread_leads`
- (fast-follow) `list_crashes`, `get_crash`

## Migration from v1.2.0

- **Emulator users on Linux**: the native AVD path is gone. Install Docker.
- **`gitd.db` premium symbols**: moved to `ghost_premium.db`. Public users won't have this and shouldn't have been calling it.
- **`OPENROUTER_MAX_TURNS`** now respected.
