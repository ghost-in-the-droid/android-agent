# Ghost in the Droid v1.3.0

_Release candidate — target ship: TBD_
_Status: DRAFT — updated as features land_

## In one line

Ghost 1.3 turns Ghost in the Droid into a full **agent harness for Android** — six LLM backends, on-device Gemma, physical-device support, honest AndroidWorld numbers. Same 38 MCP tools, same skill engine — swap the brain, keep the body.

## Highlights

- 🧠 **6 LLM backends** — Claude Code, Anthropic API, OpenRouter, Ollama, on-device Gemma, vLLM
- 📱 **On-device inference** ⭐ — Gemma runs inside the Android app itself via MediaPipe (`.task`) or llama.cpp (`.gguf`). Airplane-mode works.
- 🐳 **Docker+KVM emulator backend** — replaces broken native AVD path; works on Linux, macOS, CI
- 🔍 **Tracing tab** — per-turn traces, token accounting, tool-call visibility
- 🌐 **Web search tool** — agent can search mid-conversation
- 🔌 **Marketing jobs seam** — `POST /api/marketing-jobs/enqueue` for external orchestrators (e.g. content agents driving Ghost's posting)
- 🧭 **`lookup_lead` + `list_unread_leads`** MCP tools for marketing agents
- ⚡ **Token-efficient a11y protocol** — visible-first + diff snapshots + expiring refs; big token/latency win
- 🐛 **Crash-report tools** — capture logcat + tombstone context from any tap sequence
- 🎯 **Clean AndroidWorld submission** — score TBD, unmodified upstream harness, submitted trajectories

## Fixes worth calling out

- 🔴 **On-device tool args** now handle both flat and nested shapes (was silently dropping args from Gemma's trained shape)
- 🔴 **`Device.adb` surfaces errors** — was returning stdout with `check=False`, letting ~38 MCP tools inherit phantom-success
- 🟡 **`stop_agent` respects session boundaries** — was globally killing every claude subprocess, breaking multi-session
- 🟡 **Test suite is honest** — removed fixtures that wiped the real dev DB and assertions that passed on error
- 🟡 **Three `tap-element` code paths consolidated** — was possible for the agent to tap the wrong element because indexes weren't shared

## Cleanup / hygiene

- **`gitd/db.py` slimmed** — moved premium schema (influencer / outreach / content templates) into the private plugin where it belonged
- **Deleted ~180 LOC of dead provider code** (`_chat_claude_code` — routed elsewhere years ago)
- **OpenRouter provider** now has the full multi-turn tool-use loop (was single-shot)
- **CORS + input validation hardened** across all POST endpoints

## API surface changes

**New endpoints:**
- `POST /api/marketing-jobs/enqueue` — enqueue posting jobs from external orchestrators (forces `action='draft'` for safety)

**Provider list:**
- `GET /api/agent-chat/providers` now returns 6 providers (up from 4 in v1.2.0)

**MCP tools:**
- New: `lookup_lead`, `list_unread_leads`
- Refined: `get_elements` / `tap_element` share consistent indexing

**Breaking changes:** None expected. Migration section below covers edge cases.

## Migration from v1.2.0

- **Emulator users on Linux**: the native AVD path is gone. Install Docker and follow [Docker emulator setup](https://ghostinthedroid.com/features/emulator/) — much smoother.
- **Anyone using the raw `gitd.db` module**: the premium schema functions (`upsert_influencer`, etc.) moved to `ghost_premium.db`. Public users won't have this and shouldn't have been calling it.
- **`OPENROUTER_MAX_TURNS` env var** now respected (previously OpenRouter ran single-shot regardless).
